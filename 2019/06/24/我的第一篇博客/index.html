hexo博客搭建和主题更换

发表于 2019-06-24 | 更新于 2019-06-25 | 阅读次数：

1. hexo博客搭建

1.1 必备组件安装

1.1.1 node.js

  直接进入node.js的官网下载LTS文件，按步骤进行安装即可。安装完成后,会得到node.js本身和npm组件，npm是一个包管理工具，后续安装其他包需要这个指令。可以用以下两个指令查看版本号（能查到说明安装成功）。

1.1.2 Git

  Git的下载地址
  下载好后直接安装，根据提示安装即可。详细安装过程可以参考该博客。注意：如果电脑上没有安装好Git，在后续的操作中会出现奇怪的错误！

1.2 hexo博客搭建

1.2.1 cnpm淘宝镜像

  为了保证后面包的下载速度，需要先安装好淘宝镜像源，在powershell中输入指令：
npm install -g cnpm --registry=https://registry.npm.taobao.org
  安装完成后也可以通过cnpm -v来查看是否能用。

1.2.2 hexo框架安装

  用到之前已经安装好的cnpm来安装hexo，在命令行输入下述指令：
cnpm install -g hexo-cli
安装好后，可以用hexo -v来验证是否安装成功，如果安装成功，应该会得到如下的回复。

1.2.3 正式开始搭建hexo博客

  首先建立一个新的文件夹，这里命名为“blog”，后续所有的操作都是在这个文件夹里面。（Tips：如果后续的过程中，出现了什么意想不到或者暂时解决不了的错误，直接把该文件夹删除即可，重头再来，不怕~）   用管理员身份运行powershell，将命令行切换至该文件夹目录下，输入指令hexo init初始化博客，这时会远程连接到GitHub自动下载安装所需文件，初始化时间取决于你的网速（我自己大概需要40s左右），安装完成后会有一个提示：
INFO Start blogging with Hexo!
  blog文件夹下就会生成多个文件，其中关键的有三个文件，source文件夹用来存放博客内容，themes文件夹用于存放主题相关，_config.yml文件为站点配置文件，后续优化博客需要用到这个配置文件。

  回到blog目录下，先用hexo clean清理一下，然后再用hexo g重新生成，接着用hexo s启动本地博客。可以发现自己的博客下多了一篇刚才我们添加的新博客。

1.2.4 部署到GitHub

  首先在自己GitHub的Repositories里新建一个仓库。由于GitHub是一个免费的仓库，所以仓库名必须按照规定来（当然也可以去购买一个域名，显得比较有特点），仓库名必须是xxx.github.io，其中xxx是自己的GitHub名称，如果没按该格式命名，可能打开博客会出现404的情况。
  接着，要在blog目录下安装一个git部署的插件，输入以下指令，忽略一些警告，出现“all packages installed”时就安装好了。
cnpm install --save hexo-deployer-git

repo后面的连接就是刚才在GitHub上创建仓库时的仓库地址。
  接下来用hexo d指令将本地blog推送到GitHub上，推送成功后本地blog目下会增加一个public文件夹，远端GitHub的仓库里会多出很多文件。
  这个时候在浏览器里输入自己的仓库名xxx.github.io，就可以直接访问博客，就可以不用localhost，在任何电脑都可以打开。这样最基本的博客就搭建好了。

1.2.6 更换主题

  如果不喜欢初始的主题，可以在hexo官网上下载其他主题，hexo提供很多漂亮的主题，比如我现在使用的NexT主题，在blog目录下输入：
git clone https://github.com/theme-next/hexo-theme-next themes/next
  克隆后，themes文件夹里就多了一个next文件夹，该文件夹里包含着后续美化主题的各种配置文件。然后打开blog目录下的_config.yml文件，搜索“theme”,将默认的landscape改为next，保存后，再用hexo s打开本地博客，就会发现自己博客的主题已经改变了。同样，再用hexog和hexo d部署到GitHub上，这样用xxx.github.io打开也能看到主题变化了。
  至此，博客的搭建和主题的选择就完成了，后续可以参考其他教程继续优化自己的博客，毕竟属于自己的一个小天地，还是按照自己的意愿来修改最好了。
